Version History

Lightcurve MC conforms to version 2.0.0 of the Semantic Versioning specification. All version numbers are to be interpreted as described therein. The "User's Guide" page, not including the "Examples" section, fills the role of the public API for the program.

Version 2.3.0

Changes

  • Generic light curve code has now been moved to the kpfutils common library
  • Factored Gaussian process code into a new class, GaussianProcess
  • Factored command line parsing code into separate functions.
  • Versioning convention updated from version 2.0.0-rc.1 to version 2.0.0 of the Semantic Versioning specification.

New Features

  • Support for AA Tau light curves
  • Support for Gaussian process timescales

Bug Fixes

  • makefile is now more robust to missing directories
  • Periodic light curves now check for invalid width and width2 parameters.
  • Observations now handles exceptions correctly.
  • Observations objects are now correctly randomized.
  • Undefined values of C1 and the period will no longer be omitted from the program output. Program output now includes the fraction of runs in which C1 is defined.
  • simple_gp and two_gp light curves now work correctly when one of the input time stamps equals exactly zero
  • Corrected some compatibility issues with GCC 4.7's warnings system
  • Error messages related to parameters now use parameter names expected by the user

Version 2.2.0

Changes

  • unit_dmdt.cpp now conforms to ISO C++
  • Added test cases to test_stats.
  • Shortened all integration test scripts, cutting the testing time by a factor of five without sacrificing accuracy
  • Improved error recovery and reporting
  • Make generates fewer spurious compilations
  • Added installation instructions and user guide. The "User's Guide" page, not including the "Examples" section, will be henceforth considered the public API for the purposes of semantic versioning.
  • Corrected and standardized documentation
  • Added PDF documentation in doc/latex/refman.pdf.

New Features

  • --add keyword can now take arbitrary file names as arguments. The use of the keywords NonSpitzerNonVar, NonSpitzerVar, SpitzerNonVar, and SpitzerVar is now deprecated.
  • --stat (-s) keyword now allows run time selection of statistics to compute for each light curve
  • Support for ACF timescales and peak-finding timescales

Bug Fixes

  • Output files now give more significant digits for the simulation parameters, reducing naming conflicts between bins. Light curve dump files created with the --print keyword now use the same naming convention as the statistic output files.
  • Noise will no longer be correlated with WhiteNoise, RandomWalk, and DampedRandomWalk light curves
  • FlarePeak and FlareDip light curves now have correct parameter checking
  • ApproxEqual now tests the correct condition

Version 2.1.0

Changes

  • Added documentation for test code. Declared a new namespace, lcmc::test, containing all test suites.
  • Added test cases to test_stats.
  • Added test_wave, a unit test suite for the older periodic light curves
  • Added cmdtest.sh, an integration test suite for command line errors
  • makefile now handles include dependencies in subdirectories correctly
  • Removed information about the local GCC configuration from makefile.inc.
  • Incorporated mcio.cpp, lcsio.cpp, and kpffileio.cpp into main program rather than as an external dependency.
  • Sped up evaluation of SimpleGp and TwoScaleGp by a factor of five in the case that the model parameters are held fixed (i.e., --amp, --period, etc. are given with a range of zero length)

New Features

  • Input validation for all command line arguments
  • RandomWalk and TwoScaleGp. Added corresponding test cases to test_gp.
  • Support for command-line arguments --amp2 and --period2.

Bug Fixes

  • Bug that SimpleGp produces inconsistent output downgraded to a warning, and moved to getHalfMatrix(). See getHalfMatrix() documentation for more details.
  • Bug where deltaMBinQuantile() appeared to show the Δm median crossing half the amplitude before reaching the third-amplitude in Gaussian processes has been explained. The behavior is the result of half-amplitude crossings happening either in the poorly populated bins where Δt ≈ 0.5 day, or not at all, whereas statistical fluctuations can cause spurious third-amplitude crossings (with no corresponding half-amplitude crossing) in any Δt bin. This behavior, while odd, is a problem with long ground-based survey cadences rather than with the analysis software, and is no longer considered a bug. The behavior may be reproduced with the files tests/paradox*.*.
  • Tests in test_gp now report success on completion
  • LcBinStats::analyzeLightCurve() now propagates exception types correctly
  • Test output files now distinguish runs with identical light curve parameters but different noise levels
  • The --add command line argument can no longer be combined with either the list of julian dates or with the --noise argument
  • LcBinStats::printBinStats() will now report zero standard deviation, not NaN, for statistics that were always calculated to the same value
  • The --amp parameter for sines now represents the half-amplitude, as intended, not the amplitude
  • Light curves no longer generate negative fluxes
  • Integration test scripts run more consistently across systems. In particular, they are now designed for execution, not sourcing.
  • makefile now finds libraries on C-shell

Version 2.0.0

Changes

  • Redid the entire way information is passed to ILightCurve subclasses, in particular rewriting the interface of the ParamList class from scratch and moving it outside the ILightCurve class.
  • Redid ILightCurve interface to add support for stochastic light curves.
  • Deleted ILightCurve::modelParams()
  • Cleaned up documentation.
  • From now on, version numbers will conform to version 2.0.0-rc.1 of the Semantic Versioning specification.

New Features

  • Replaced period-amplitude command-line interface with one that allows the user to specify an arbitrary set of parameters, but at the cost of evaluating only a single bin per run
  • Implemented WhiteNoise, DampedRandomWalk, and SimpleGp.
  • Support for ΔmΔt timescales

Bug Fixes

  • Fixed: C1 will now be calculated correctly in magnitude space for light curves where the measured (noisy) flux is negative.
